	2)什么程度?		-> 业务
	
		我的便签-> 不用记录日志 直接对数据进行CUDR操作 查看本人信息
		发送通知-> 
			员工 可以发送给本部门的员工和经理 甚至可以发送给高管 需要记录日志                    --对消息表进行增删改查操作 
			经理 可以发送给本部门的员工 也可以发送给其他部门经理  可以发送给高管 需要记录日志
			高管 可以发送给公司任何员工或经理 需要记录日志
		通知记录
			保存自己发送的信息 并 获取对方发送的信息 分页排序查找 可以删除
			
		请假
			员工 请假信息直接投递到本部门经理（如果经理暂缺 直接发送给高管）	需要记录日志
			经理 请假信息直接投递到高管	需要记录日志
		请假记录
			保存自己发送的请假信息和状态 分页排序查找
		审核列表
			记录自己下属发送的请假申请（待审核排前）
		请假审核 
			经理 对请假信息进行更新操作 需要记录日志
			高管 对请假信息进行更新操作 需要记录日志	
			
		部门管理 -> 高管
			新增部门	-> 添加(部门负责人空缺) 
			任命负责人	-> 
					部门负责人空缺 -> 更新角色（经理） 更新负责人
					部门负责人换位 -> 原负责人更新角色（员工）	新负责人更新角色（经理） 更新负责人
			解散部门 	-> 遣散本部门员工
			
		
		日志记录(确保有效操作)
			序号  	用户id  用户名 角色 所属部门	操作	时间	内容	备注
			
		用户管理
			用户表增加角色字段 | 保留 用户表 用户角色表 角色
		
		角色管理
			角色表 角色权限 权限表
